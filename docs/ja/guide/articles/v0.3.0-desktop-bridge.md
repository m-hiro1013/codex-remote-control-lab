# v0.3.0 Desktop Bridge Walkthrough

![Codex Remote Control Lab v0.3.0](/release-header-v0.3.0.svg)

v0.3.0 は、Codex Remote Control Lab が単なる phone page ではなく、compact な desktop control surface として使えるようになったリリースです。safety model はそのまま維持しながら、作業確認、artifact open、thread 切り替えで operator が期待する部品を UI に揃えました。

## Hand-edited HTML ではなく保守できる shell へ

shell は `src/ui/Shell.jsx` から始まります。`scripts/build-ui.mjs` が React source を `public/index.html` に render し、`npm run check` は生成済み HTML が古いと失敗します。

これは、UI の構造が増えたからこそ重要です。sidebar、composer、model menu、panel tabs、accessibility label を React source 側に集めることで、直接 HTML 編集による drift を避けやすくしています。

## Chat しながら review する

bridge は conversation 内に review digest を表示できます。まず working tree を見て、変更がなければ latest commit に fallback します。digest には changed files、additions / deletions、artifact panel で開ける file の quick-open card が出ます。

これは本格的な code review の代替ではありません。ただ、phone や second browser から、terminal に戻る前に「何が変わったか」を掴めるようになります。

## Workspace と artifact panel

右 panel は artifact、workspace、review mode に分かれています。workspace listing は意図的に bounded です。depth と件数を制限し、directory を先に並べ、`.env`、`.phone-token`、`.codex-home*`、`.uploads`、`node_modules` のような local-only path は除外します。

左 thread rail と右 panel は resize できます。幅は browser local storage に保存されるため、毎回 bridge を開くたびに operator setup を作り直す必要がありません。

## より安全な debug mode

local UI debug は tokenless でできますが、tokenless LAN sharing を default で安全だとは扱いません。`PHONE_DEBUG_NO_TOKEN=1` は bridge を `127.0.0.1` に bind し、localhost URL を表示します。private LAN で tokenless debug を出す場合は、operator が明示的に `PHONE_DEBUG_BIND=lan` を追加します。

この分離により、通常の LAN use は token-protected のままです。trusted-network debug exception は、operator が意図した場合だけ有効になります。
