# v0.4.0 リリースノート

![Codex Remote Control Lab v0.4.0](/release-header-v0.4.0.svg)

v0.4.0 は `v0.3.0` から `release/v0.4.0` branch までの変更をまとめたリリースです。実際の mobile screen で phone bridge を安定して扱えるようにしつつ、実行中の provider、turn state、利用できる skill や plugin を operator が読み取りやすい surface に整えています。

## ハイライト

- mobile の artifact card は、chat と review digest のどちらからでも意図した artifact を開けるようになりました。狭い画面や text node を tap した場合も対象を見失いません。
- composer は iOS focus zoom を避けるため、browser zoom 自体を無効化せず、mobile-safe な input font size を維持します。
- slash skill discovery は実際に導入済みの plugin skill file を読み、skill lookup が一時失敗した場合に retry でき、mobile の全角 slash 入力も正規化します。
- plugin panel は導入済み/有効な entry を既定表示にし、phone 上で marketplace-only entry に埋もれないようにしました。
- thread switching は選択先の履歴が ready になるまで直前の表示を維持し、古い thread の ready message が後から来ても上書きしません。
- reconnecting、interrupting、approval、completion、failure などの実行状態を bridge UI がより明確に表示します。

## Bridge Reliability

- Codex stream の reconnect 系 error は raw payload を chat に流さず、bridge status として正規化します。
- browser bridge が切断された後の background thread polling は同じ disconnect error の連続表示を抑えます。短い app-server restart や token mismatch でも会話が読みにくくなりません。
- turn がまだ存在しない段階の interrupt は turn 作成後まで保留され、operator が中断したときは待機中の送信 queue も破棄します。
- 任意設定の通知 channel では、Codex turn の completed、failed、approval waiting を task notification として送れます。

## Provider And Status Surfaces

- `npm run phone:claude` は `PHONE_AGENT_PROVIDER=claude` を設定し、同じ browser bridge UI を Claude Code 向けに起動します。
- Claude mode は同じ workdir の Claude Code JSONL session を sidebar に読みます。一方で Codex 専用の plugin lookup、live approval callback、history sync は無効のままです。
- Codex rate-limit 表示は `PHONE_CODEX_RATE_LIMIT_REFRESH_COMMAND="node scripts/read-desktop-rate-limits.js"` で任意に有効化できます。
- rate-limit reader は local Codex auth file を使って remaining percentage と reset metadata だけを正規化し、bridge 側は表示用 snapshot だけを cache します。
- workspace metadata refresh は cache され、repo、branch、location を status strip に出すために Git を何度も同期的に呼び続けません。

## Docs, Evidence, And Public Safety

- mobile artifact tap、mobile slash skill insertion、実際の installed skill discovery、iOS input zoom protection の proof asset を追加しました。
- CI は Node test suite の前に Playwright 用 Chromium を install します。
- `CONTRIBUTING.md`、`docs/guide/contributing.md`、`docs/ja/guide/contributing.md` に公開安全な upstream PR path を追加しました。
- `tmp/` 配下の campaign/release scratch file は tracking から外し、local rate-limit / workspace cache file も `.gitignore` に追加しました。
- `AGENTS.md` には UI/browser behavior を script-based Playwright と `browser-use:browser` の両方で確認する expectation を記録しました。
